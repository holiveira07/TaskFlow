# HISTÓRICO DO PROJETO

## Origem

O projeto TaskFlow nasceu durante o curso de Engenharia de Software.

Inicialmente o objetivo era apenas praticar JavaScript.

Durante o planejamento do curso foi decidido que aprender apenas a linguagem não seria suficiente.

O projeto passou então a ter um objetivo muito maior:

Construir um sistema completo enquanto todas as competências necessárias para atuar como Engenheiro de Software fossem aprendidas.

Desde esse momento o TaskFlow deixou de ser apenas um exercício e passou a ser o projeto principal de toda a jornada.

# PRINCIPAIS DECISÕES TOMADAS

## Decisão 001

Não desenvolver o sistema antes de compreender os fundamentos da programação.

Motivo:

Construir software sem dominar a linguagem gera dependência de copiar código e dificulta a evolução do projeto.

Aprender JavaScript profundamente antes de iniciar Node.js.

Motivo:

Node.js utiliza JavaScript.

Quanto melhor for o domínio da linguagem, mais simples será aprender backend.

---

## Decisão 003

Criar documentação desde o início.

Motivo:

O projeto deverá simular um ambiente profissional.

Toda decisão importante deverá ficar registrada.

---

## Decisão 004

Versionar absolutamente tudo utilizando Git.

Motivo:

Aprender versionamento faz parte da formação do engenheiro de software.

---

## Decisão 005

Utilizar GitHub desde o primeiro dia.

Motivo:

Todo o desenvolvimento deverá ficar registrado.

O GitHub será utilizado também como portfólio.

---

## Decisão 006

Não utilizar projetos fictícios desconectados.

Todo conteúdo aprendido deverá, sempre que possível, contribuir para o TaskFlow.

# PRINCÍPIOS DE ENGENHARIA DE SOFTWARE

O desenvolvimento do TaskFlow seguirá os seguintes princípios.

---

# Clareza antes de complexidade

A solução mais simples que resolve corretamente o problema deve ser priorizada.

Não criar complexidade desnecessária.

---

# Código para pessoas

Código será escrito pensando em manutenção.

Um código que funciona mas ninguém entende não é considerado uma boa solução.

---

# Responsabilidade única

Cada módulo, função ou classe deverá possuir uma responsabilidade clara.

Evitar componentes que fazem muitas coisas ao mesmo tempo.

---

# Evolução incremental

O sistema será construído em pequenas entregas.

Não tentar criar todo o sistema de uma vez.

# Clean Code

Objetivo:

Criar código fácil de ler e manter.

Princípios:

- nomes claros;
- funções pequenas;
- evitar duplicação;
- organização.

---

# SOLID

Objetivo:

Criar sistemas mais flexíveis.

Princípios:

## S — Responsabilidade Única

Uma classe deve possuir apenas uma responsabilidade.

---

## O — Aberto/Fechado

Código deve permitir evolução sem alterar constantemente o que já funciona.

---

## L — Substituição de Liskov

Objetos derivados devem poder substituir seus objetos base.

---

## I — Segregação de Interfaces

Interfaces menores são melhores que interfaces gigantes.

---

## D — Inversão de Dependência

Módulos de alto nível não devem depender diretamente de detalhes.

# SEGUNDA ETAPA — MANIPULAÇÃO DE ARRAYS

Esta etapa teve como objetivo desenvolver a capacidade de trabalhar com coleções de dados.

Foram estudados:

---

## length()

Objetivo:

Descobrir a quantidade de elementos de um array.

Aprendizado:

Arrays possuem posições iniciando em zero.

---

## filter()

Objetivo:

Criar um novo array contendo apenas elementos que atendem uma condição.

Aprendizado:

O método não altera o array original.

---

## map()

Objetivo:

Transformar elementos de um array.

Aprendizado:

O método percorre cada elemento e retorna um novo array.

Exemplo de aplicação:

Transformar objetos de tarefas em títulos.

---

## find()

Objetivo:

Encontrar um único elemento.

Aprendizado:

Retorna o primeiro elemento encontrado.

---

## join()

Objetivo:

Transformar array em string.

Exemplo:

Lista de tarefas transformada em texto.

---

# TERCEIRA ETAPA — MÉTODOS QUE MODIFICAM ARRAYS

Nesta etapa foi estudada a diferença entre métodos que criam novos arrays e métodos que alteram o array existente.

---

## push()

Aprendizado:

Adiciona elementos no final.

Modifica o array original.

---

## pop()

Aprendizado:

Remove elementos do final.

---

## unshift()

Aprendizado:

Adiciona elementos no início.

---

## shift()

Aprendizado:

Remove elementos do início.

---

## splice()

Conteúdo concluído:

O método splice foi estudado completamente.

Aprendizados:

- funciona em qualquer posição;
- recebe índice inicial;
- recebe quantidade de remoção;
- pode inserir elementos;
- pode substituir elementos;
- pode remover vários elementos.

Exemplos compreendidos:

Inserção:

```javascript
array.splice(posicao, 0, elemento)

Substituição:
array.splice(posicao, quantidade)

Remoção:

array.splice(posicao, quantidade)



FORMA DE APRENDIZADO CONSOLIDADA

Primeiro:

Entender o raciocínio do computador.

Depois:

Escrever código.

Depois:

Aplicar em problemas reais.

Essa abordagem deverá continuar durante todo o curso.
```
